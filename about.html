About

This is the project page for mdsh (https://sc0ttj.github.io/mdsh/).

What is mdsh?

A static site generator, which produces a simple blog.

The name mdsh refers to the custom Markdown + Shell format used to write blog posts.

Features

  • Written in Bash shell script
  • Easy setup - just clone the repo, edit the .site_config file.
  • Use a mix of Markdown and shell code to write blog posts!
  • Use your terminal - no text editor or IDE required.
  • Automatically generates index and listings pages for categories, tags, and more.
  • Builds a fast blog with great SEO and mobile support
  • Supports fullscreen, native app experience if added to Homescreen (on mobile)
  • Uses Google Fonts to choose custom web fonts
  • Uses Unpkg.com to manage JavaScript libraries/dependencies
  • Uses Git for saving, managaging and reverting versions and deployments
  • Uses GitHub Pages for fast, free hosting

Usage

  1. Clone the repo: git clone https://github.com/sc0ttj/mdsh
  2. Enter project dir: cd mdsh
  3. Use GitHub Pages for hosting: git checkout -b gh-pages
  4. Edit site settings: $EDITOR .site_config
  5. Build homepage: ./create_page > index.html
  6. Create your first post:./create_post (and follow the on-screen instructions)
  7. ^ after you create a post, all site index pages and menus will be rebuilt.
  8. To re-build all pages: ./updates_pages -all
  9. To publish: git add .; git commit; git push origin gh-pages;

Advanced Usage

To add a page to the site header menu:

Create a .html page in the root dir - all html files in the root dir are automatically included in the site header menu.

To rebuild an HTML page from an existing markdown document:

./create_page path/to/file.md > output_file.html

To create custom HTML pages:

# create a variable containing some HTML
my_custom_html="<h1>Some header</h1>
<p>Some content</p>"

# pass the html to create_page, redirect to our new html file
./create_page "$my_custom_html" > my-custom-page.html

To edit site layouts or HTML:

Edit the HTML inside create_page or the CSS in assets/css/main.css, there are some functions in there which produce the site HTML

Delete a post/page

Delete any files you no longer want published ( .html, .md and .mdsh files), and then run ./update_pages -all.

Unpublish a post/page

Comment out the relevant post in posts.csv, and run ./update_pages -all. This will hide the page from site indexes and listings.
